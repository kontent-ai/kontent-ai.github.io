# Kontent.ai developer resources

Guidelines for building and maintaining open-source projects for Kontent.ai - SDKs, tools, and integrations. Start here, then jump to the section you need.

## What's here

- **[Publishing checklist](./Checklist-for-publishing-a-new-OS-project.md)** - everything a new repository needs before it goes public; start from our [`repo-template`](https://github.com/kontent-ai/repo-template).
- **[Naming conventions](./Naming-conventions.md)** - how to name repositories, namespaces, and packages.
- **[Building SDKs & tools](./Building-SDKs-and-tools.md)** - tracking headers, expected SDK functionality, and backward-compatibility rules.
- **[Code style](./code-style/code-style.md)** - Kontent.ai-specific conventions for .NET and JavaScript/TypeScript.
- **[Maintainer duties & SLAs](./Duties-of-a-Repository-Maintainer.md)** - what owning a repository involves, including security remediation SLAs.
- **[Resources](./Related-Resources.md)** - SDKs, documentation, and other useful links.

New here? Skim the [publishing checklist](./Checklist-for-publishing-a-new-OS-project.md), then [Building SDKs & tools](./Building-SDKs-and-tools.md). Releasing to a package manager? Follow [SemVer](https://semver.org/).
